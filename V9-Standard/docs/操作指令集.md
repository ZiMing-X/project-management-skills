## 一、使用说明

### 频率分级标准

| 频率 | 标识 | 含义 |
|:---|:---:|:---|
| **常用** | ⭐ | 日常操作中至少每周使用1次，建议新手优先掌握 |
| **不常用** | 📌 | 特定场景下使用，按需调用，不需要死记硬背 |

### 阅读建议

| 身份 | 建议阅读范围 |
|:---|:---|
| 新手 | Base 常用部分 → 直接按场景导航操作 |
| 日常用户 | Standard 所有常用 + Base全部 |
| 重度用户 | 升级至 V9-Full |

### 指令格式说明

每条指令包含：
- **指令名称**：技能文件名
- **作用**：一句话说明做什么
- **触发词**：在对话框中输入即可激活
- **场景示例**：真实使用案例
- **权限**：技能执行所需的工具权限
- **安全/约束**：使用时需要注意的限制

## 三、V9-Standard 新增技能详解

### ⭐ 常用指令（3个）

#### 06 context-survival — 上下文生存

| 项目 | 内容 |
|:---|:---|
| **作用** | 对抗上下文压缩导致的记忆丢失，自动写micro-log + PENDING.md待办保护 |
| **触发词** | 自动触发，无需显式调用 |
| **权限** | Read, Write, Edit, Bash |

**场景示例**：

```
场景1：无需手动操作
→ 写 @todo 笔记或 FIXME 标记时自动触发
  AI自动将关键待办写入 PENDING.md

场景2：检查当前保护状态
→ 查看 .workbuddy 目录下的 PENDING.md
```

**特点**：自动触发，写前检查 PENDING.md 条目≤20，超限告警

---

#### 07 window-aware — 窗口感知

| 项目 | 内容 |
|:---|:---|
| **作用** | 全局被动监控Token窗口消耗，Session启动时自动加载 |
| **触发词** | 自动触发 |
| **权限** | Read, Write, Bash |

**场景示例**：

```
场景1：全程无感运行
→ 🟢 ≤8%：AI静默运行，无需关注
→ 🟡 >8%：AI提醒"窗口消耗已超过8%"
→ 🔴 >12%：AI建议"已达警戒线，建议收工或清理上下文"
```

**阈值**：
- 🟢 ≤8%：静默运行
- 🟡 >8%：提醒用户
- 🔴 >12%：建议收工

---

#### 08 workbuddy-debug-helper — 调试助手

| 项目 | 内容 |
|:---|:---|
| **作用** | WorkBuddy框架级BUG（B6断联/ACP拦截/窗口耗尽等）的应对方案合集 |
| **触发词** | `WorkBuddy报错` `BUG处理` `断联处理` `ACP拦截` `window-aware失败` `上下文丢失` `记忆丢失` `自动化失败` |
| **权限** | Read, Write |

**场景示例**：

```
场景1：会话突然断联（B6）
→ "断联处理" 或 "B6"
  AI提供断联恢复步骤

场景2：ACP沙箱拦截Python执行
→ "ACP拦截"
  AI提示使用 acp-proxy-bypass 技能或检查沙箱配置

场景3：session-manager 报错
→ "session-manager失败"
```

#### 15 prompt-advisor — 提示词顾问

| 项目 | 内容 |
|:---|:---|
| **作用** | 分析提示词质量，检测语义冲突、冗余内容，给出优化建议 |
| **触发词** | `提示词顾问` `需求澄清` `分析提示词` `审查提示词` `检查提示词质量` `帮我检查提问完整性` |
| **权限** | Read, Write, Edit |

**场景示例**：

```
场景1：不确定自己的提问是否清晰
→ "分析提示词"（将你的提问附上）
  AI检测是否有模糊/冲突/冗余

场景2：需要优化工作提示
→ "提示词顾问"
```

**三种模式**：
| 模式 | 说明 |
|:---|:---|
| A 集成审查 | 语义分析+冲突检测+冗余发现 |
| B 专家模拟 | 基于历史给出方向性建议 |
| C 智能需求澄清 | 自动检测信息完整性并推测补充 |
