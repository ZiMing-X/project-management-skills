# 项目管理技能体系 - 分享版

## 这是什么

一套基于 Markdown 文件的项目管理方法，让 AI 能够在每次新会话中自动理解项目状态、追踪任务进展。

## 核心思想

1. **文档即架构** — 项目的一切规则、状态、边界都沉淀在 Markdown 文件中
2. **约束即自由** — 明确的项目边界和白名单让 AI 专注不越界
3. **技能即资产** — 可复用的 AI 技能文档，积累形成组织级知识库

## 快速开始

1. 复制本文件夹到你的项目根目录
2. 用编辑器全局搜索 `{`，替换所有占位符为你的实际路径
3. 从 `skills/00-技能全景图.md` 开始了解体系结构
4. 使用 `project-workspace` 初始化你的第一个项目

## 文件说明

| 文件 | 作用 |
|------|------|
| `skills/00-技能全景图.md` | 体系总览与架构 |
| `skills/01-project-workspace.md` | 项目日常操作核心 |
| `skills/02-project-tracker.md` | 事件记录与周报 |
| `skills/03-project-administrator.md` | 项目注册与注销 |
| `skills/04-project-config-editor.md` | 项目配置文件编辑 |
| `skills/05-check-project-health.md` | 健康度检查 |
| `skills/XX-已归档技能.md` | 归档规则参考 |
| `templates/` | 通用项目模板（可直接用于初始化） |

## 适用场景

- 个人知识库管理
- 小团队项目协作
- AI 辅助开发工作流
- 需要 AI 理解项目边界和白名单的项目

## 注意事项

- 这是一个思想示范，请根据实际需求定制
- 所有个人路径已替换为占位符 `{xxx}`
- 不依赖任何特定 AI 平台
- 使用前请先替换所有占位符
