## 检查项清单

### 🔴 阻断性（必须立即修复）

| ID | 检查项 | 检查方法 | 修复建议 |
|----|--------|---------|---------|
| H-01 | project.md 文件缺失 | 检查 `{项目根目录}\project.md` 是否存在 | 使用 project-workspace 初始化项目骨架 |
| H-02 | PROJECT.md 缺少必要章节 | 检查是否包含"基本信息"、"核心任务白名单"等必要章节 | 补充缺失的章节 |

### 🟡 警告（建议尽快修复）

| ID | 检查项 | 检查方法 | 修复建议 |
|----|--------|---------|---------|
| W-01 | ops/文件含未替换占位符 | 搜索ops/目录下含`{{`的.md文件 | 手动替换占位符，或使用project-workspace重新生成 |
| W-02 | CHANGELOG日期不连续 | 提取CHANGELOG.md中所有`## YYYY-MM-DD`格式的日期，验证连续性 | 补齐缺失日期的日志条目 |
| W-03 | PROJECT.md模块表格格式异常 | 检查Markdown表格是否有未闭合的`|`、列数不一致的行 | 手动修复PROJECT.md表格格式 |
| W-04 | 项目管理体系版本非最新 | 读取PROJECT.md中"项目管理体系版本"字段，与SKILLS/00-技能全景图.md对比 | 执行project-workspace的版本升级流程 |
| W-05 | memory/state.md 缺失 | 检查活跃项目的 `memory/state.md` 是否存在 | 使用 project-workspace 初始化 memory 目录 |

### 🔵 信息（供参考）

| ID | 检查项 | 检查方法 |
|----|--------|---------|
| I-01 | 所有项目CHANGELOG日期无间断 | 报告各项目最后日志日期 |
| I-02 | PROJECT.md状态与实际输出一致 | 抽样检查✅完成状态的模块其输出文件是否真实存在 |
| I-03 | project-registry与实际目录一致 | 对每个注册项目验证根目录是否真实存在 |
| I-04 | 项目管理体系版本为最新 | 验证PROJECT.md中"项目管理体系版本"与最新体系版本一致 |
| I-05 | 项目日志连续性 | 检查活跃项目的 `项目日志/` 目录是否有近7天内的 .md 记录 |

## 检查流程

```
Step 1: 读取注册表
    ↓ 读取project-registry.md
Step 2: 对每个项目执行检查
    ↓ 逐一执行检查项扫描
Step 3: 汇总报告
    ↓ 生成三级报告
```

## 交付物与注意事项

### 成果说明

本技能的交付物是**三级健康度报告**：
- 🔴 阻断性问题：必须立即修复，否则阻止下游流程
- 🟡 警告问题：建议尽快修复
- 🔵 信息：仅供参考

### 适用场景

✅ **最佳适用**：
- 定期项目体检（建议每周或每个项目阶段完成后）
- 上线前的最终检查
- 发现问题后的诊断

❌ **不适用**：
- 修复问题（本技能只读不写）
- 实时监控（建议定期运行）

### 使用注意事项

| 注意事项 | 说明 |
|---------|------|
| ⚠️ 纯读取操作 | 本技能不会修改任何文件，只读取并报告问题 |
| ⚠️ 范围限定 | 仅扫描注册表中的项目，不会扫描注册表外的内容 |
| 💡 优先修复🔴问题 | 阻断性问题不修复会阻止下游流程 |
| 💡 定期检查 | 建议每周或每个项目阶段完成后运行一次 |
