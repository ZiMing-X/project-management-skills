## 二、配置层技能

### project-administrator（项目配置管理）

| 属性 | 说明 |
|------|------|
| **定位** | 项目创建/删除/注册的入口 |
| **核心能力** | 新增项目、删除项目（仅移除注册表）、整机备份配置 |
| **备份机制** | 强制整机备份到 `{备份根目录}` |
| **安全特性** | 删除操作不触碰任何磁盘文件，只移除注册表条目 |

**触发场景**：
- 项目初始化场景：需要创建新项目时
- 项目注销场景：需要移除项目注册时
- 备份场景：需要备份所有配置文件时

---

### project-config-editor（配置文件编辑）

| 属性 | 说明 |
|------|------|
| **定位** | project.md 文件的细粒度编辑 |
| **核心能力** | 查看/追加/删除白名单、修改基本信息、调整排除项 |
| **备份机制** | 单文件轻量备份到 `{备份根目录}/单文件备份/` |
| **操作范围** | 仅操作 project.md，不得触碰注册表或其他文件 |

**触发场景**：
- 查看配置场景：需要查看项目白名单或基本信息时
- 修改配置场景：需要调整白名单、排除项或基本信息时

## 三、文档层技能

### project-workspace（统一项目空间管理）⭐

| 属性 | 说明 |
|------|------|
| **定位** | 项目骨架创建与维护的统一入口 |
| **核心能力** | 初始化项目骨架、模块状态更新、生成操作指令模板 |
| **模板模式** | 通用模板（可扩展自定义模板） |
| **前置检查** | 状态更新前强制验证输出文件存在性 |
| **自动推进** | 任务完成时自动推进"下一个行动" |

**触发场景**：
- 初始化场景：需要创建新项目骨架时
- 模块管理场景：需要添加或更新模块状态时
- 任务汇报场景：任务完成需要更新进度时
- 操作指令场景：需要生成操作文档模板时
- 里程碑记录场景：需要记录重要节点时

**模板结构**：
```
通用项目：
├── README.md
├── PROJECT.md
├── CHANGELOG.md
├── docs/
│   ├── requirements.md
│   └── architecture.md
└── ops/
    └── OP-README.md
```

## 四、执行层技能

### project-tracker（项目事件记录）

| 属性 | 说明 |
|------|------|
| **定位** | 项目事件记录助手 |
| **核心能力** | 生成项目事件日志、进度报告、待办清单 |
| **行为原则** | 被动响应、模板强制、范围锁定 |

**触发场景**：
- 事件记录场景：需要记录每日/每次的进展时
- 周报场景：需要生成本周/本月进度报告时
- 项目状态场景：需要查看项目整体状态时
- 跨项目联动场景：需要在多个项目间建立事件关联时
- 版本升级场景：需要记录体系版本升级时

## 五、质量层技能

### check-project-health（项目健康度检查）

| 属性 | 说明 |
|------|------|
| **定位** | 项目健康度检查工具 |
| **核心能力** | 三级报告生成（🔴阻断/🟡警告/🔵信息） |
| **行为原则** | 纯读取操作，不修改任何文件 |

**触发场景**：
- 体检场景：需要检查项目整体健康状态时
- 验收场景：项目上线前需要确认文档完整性时
- 诊断场景：发现问题后需要定位问题根源时
