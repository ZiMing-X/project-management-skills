# 02. project-tracker（项目事件记录）

> **技能文件**：`{用户技能目录}\project-tracker\SKILL.md`
> **定位**：执行层（操作层）
> **版本号**：v1.5.0（分享版）
> **最后更新**：YYYY-MM-DD

---

## 基本信息

| 属性 | 说明 |
|------|------|
| **技能名称** | project-tracker |
| **定位** | 项目事件记录助手 |
| **核心能力** | 生成项目事件日志、进度报告、待办清单 |
| **行为原则** | 被动响应、模板强制、范围锁定 |

---

## 核心铁律

| 铁律 | 说明 |
|------|------|
| **范围锁定** | 任何操作前，必须先读取并理解 `{项目根目录}/project.md` 文件 |
| **模板强制** | 所有输出必须严格遵循固定模板，不得添加、删除或修改任何章节标题 |
| **被动响应** | 只负责根据用户指令填充模板内容，不主动扩展范围 |
| **静默执行** | 除非用户明确要求，否则不输出任何解释性文字 |

## 项目边界获取

1. **接收项目标识**：用户指令中可能包含 `项目=别名` 或 `项目路径=路径`
2. **别名解析**：读取项目注册表，查找对应别名获取真实路径
3. **读取项目定义**：根据确定路径，读取 `{项目根目录}/project.md`

## 执行流程

```
1. 解析项目标识，获取 {项目根目录} 路径
2. 读取 {项目根目录}/project.md，建立严格的边界认知
3. 读取用户指定的任何相关文件（如任务的 memory.md）
   但仅提取与白名单任务ID相关的内容
4. 填充对应的模板，确保所有内容都在边界内
5. 输出填充后的文档
   如需保存，根据用户指令将输出写入指定文件
```

**强制检查点**：在输出前，必须对每一项内容进行自问：
> "这条信息是否直接关联到 `project.md` 中定义的项目核心任务？"
> 若否，则删除。

## 交付物与注意事项

### 成果说明

本技能的交付物是**结构化项目跟踪文档**：
- 事件进程记录：记录每个会话/事件的核心指标、完成操作、待办事项
- 周报：汇总本周进展、任务执行情况、下周计划

### 适用场景

✅ **最佳适用**：
- 正式项目事件记录（区别于project-workspace的轻量日志）
- 周期性进度汇报
- 项目里程碑记录

❌ **不适用**：
- 轻量级变更日志（使用project-workspace的CHANGELOG.md）
- 非项目相关的通用记录
- 项目范围外的任务跟踪

### 使用注意事项

| 注意事项 | 说明 |
|---------|------|
| ⚠️ 范围锁定是核心原则 | 严格限定在project.md定义的项目边界内 |
| ⚠️ 白名单任务ID | 只处理白名单内的任务ID，之外的一律忽略 |
| ⚠️ 模板格式不可更改 | 所有章节标题固定，不得随意添加或删除 |
| 💡 与project-workspace互补 | 轻量日志由project-workspace处理；正式报告由本技能处理 |
