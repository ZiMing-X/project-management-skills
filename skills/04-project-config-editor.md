# 04. project-config-editor（配置文件编辑）

> **技能文件**：`{用户技能目录}\project-config-editor\skill.md`
> **定位**：配置层（宪法层）
> **最后更新**：YYYY-MM-DD

---

## 基本信息

| 属性 | 说明 |
|------|------|
| **技能名称** | project-config-editor |
| **定位** | 项目配置文件编辑助手 |
| **核心能力** | 查看/追加/删除白名单、修改基本信息、调整排除项 |
| **备份机制** | 单文件轻量备份到 `{备份根目录}/单文件备份/` |

---

## 核心铁律

| 铁律 | 说明 |
|------|------|
| **修改前必须备份** | 任何修改操作前，必须先备份当前project.md文件 |
| **单文件备份** | 备份到 `{备份根目录}/单文件备份/` |
| **只操作project.md** | 不得触碰注册表、Skill文件或其他项目无关文件 |
| **备份文件名格式** | `{项目别名}_project_YYYYMMDD_NNN.md` |

---

## 全局路径常量

| 路径 | 说明 |
|------|------|
| 注册表文件 | `{用户根目录}\.workbuddy\project-registry.md` |
| 单文件备份目录 | `{备份根目录}/单文件备份/` |

---

## 项目定位与路径获取

```
1. 解析项目标识
   ↓ 用户指令中提取 项目=别名 或 项目路径=... 或明确别名
2. 读取注册表定位路径
   ↓ 读取project-registry.md，查找别名对应的 {项目根目录}
3. 确定目标文件
   ↓ {项目根目录}\project.md
4. 读取当前内容
   ↓ 使用Read工具读取全部内容
```

## 操作 A：查看项目配置

### 执行流程

```
1. 获取project.md路径并读取内容
   ↓
2. 根据用户指定提取对应段落
   ↓
3. 格式化输出
```

### 提取范围

| 选项 | 说明 |
|------|------|
| 基本信息 | `## 基本信息` 下的所有键值对 |
| 白名单 | `## 核心任务白名单` 下的所有列表项 |
| 排除项 | `## 严格排除项` 下的所有内容 |
| 全部 | 返回整个文件内容 |

### 指令示例

```
@project-config-editor 项目=myproj 查看白名单
@project-config-editor 项目=myproj 查看基本信息
@project-config-editor 项目=myproj 查看全部
```

## 操作 B：追加白名单任务

### 执行流程

```
1. 获取project.md并读取
   ↓
2. 备份当前文件
   ↓
3. 定位白名单段落，在末尾追加一行
   ↓
4. 保持原有缩进格式
   ↓
5. 写回文件
   ↓
6. 输出摘要
```

### 追加格式

```markdown
- `新任务ID` (说明)
```

### 指令示例

```
@project-config-editor 项目=myproj 追加白名单：automation-30 (新增任务)
```

## 与 project-administrator 的区别

| 对比项 | project-administrator | project-config-editor |
|--------|---------------------|---------------------|
| **操作粒度** | 项目级别（新增/删除整个项目） | 文件级别（修改project.md内容） |
| **备份范围** | 整机备份（所有配置文件） | 单文件备份（仅目标project.md） |
| **操作对象** | 注册表 + 项目组project.md | 仅project.md |
| **使用场景** | 新项目启动、废弃项目注销 | 调整白名单、修改基本信息 |
| **触发场景** | 新增项目、删除项目 | 查看、追加、删除、更新 |

---

## 交付物与注意事项

### 成果说明

本技能的交付物是**project.md文件的变更**：
- 查看：返回指定段落的内容
- 修改：追加/删除/更新指定内容 + 单文件备份

### 适用场景

✅ **最佳适用**：
- 调整项目的白名单任务列表
- 修改项目基本信息（如名称、存档路径）
- 管理排除项（禁止扫描的目录、禁止引用的任务前缀）

❌ **不适用**：
- 新增整个项目（使用project-administrator）
- 删除整个项目（使用project-administrator）
- 批量修改多个配置文件

### 使用注意事项

| 注意事项 | 说明 |
|---------|------|
| ⚠️ 单文件备份是强制前置 | 任何修改操作前必须先备份 |
| ⚠️ 只能操作project.md | 不得触碰注册表或其他文件 |
| ⚠️ 别名必须有效 | 如果使用了不支持的别名，提示检查注册表 |
| ⚠️ 保持格式一致 | 追加/删除时保持与原有格式一致 |
