# 03. project-administrator（项目配置管理）

> **技能文件**：`{用户技能目录}\project-administrator\SKILL.md`
> **定位**：配置层（宪法层）
> **最后更新**：YYYY-MM-DD

---

## 基本信息

| 属性 | 说明 |
|------|------|
| **技能名称** | project-administrator |
| **定位** | 项目配置管理助手 |
| **核心能力** | 新增项目、删除项目（仅移除注册表）、手动备份配置 |
| **安全特性** | 操作前强制备份、删除不触碰磁盘文件 |

## 操作一：新增项目

### 输入参数

| 参数 | 说明 | 必填 |
|------|------|------|
| 别名 | 短标识，如 `myproj` | ✅ |
| 全称 | 完整名称 | ✅ |
| 项目根目录 | 绝对路径 | ✅ |
| 加入项目组 | 是/否；若"是"还需指定项目组别名 | ❌ |
| 白名单任务列表 | 格式 `ID (说明)` | ✅ |
| 状态 | 默认 `🟢 活跃` | ❌ |

### 执行流程

```
1. 备份配置（强制）
   ↓ 创建备份文件夹
2. 读取注册表，检查别名是否重复
   ↓ 若重复则提示中止
3. 读取项目组project.md（如加入项目组）
   ↓ 统计子项目数量
4. 在注册表末尾追加一行
   ↓ 更新project.md
5. 若加入项目组，在项目组project.md末尾追加子项目段落
   ↓ 输出变更摘要
```

### 注册表新行格式

```markdown
| `myproj` | 我的新项目 | `{项目根目录}` | 🟢 活跃 | 备注或留空 |
```

### 子项目project.md模板

```markdown
# 项目定义：【{全称}】

> **最后更新**：{当前日期 YYYY-MM-DD}

## 基本信息
- **项目名称**：{全称}
- **项目根目录**：`{项目根目录}`
- **项目文档存档路径**：`{项目根目录}项目日志\`

## 核心任务白名单
- `任务ID` (说明)

## 严格排除项
- **禁止扫描的目录**：`{项目根目录}测试数据\`、`{项目根目录}归档\`
- **禁止引用的任务ID前缀**：`test-*`、`deprecated-*`
- **禁止讨论的话题**：其他非本项目相关任务
```

---

## 操作二：删除项目

### 执行流程

```
1. 备份配置（强制）
   ↓
2. 读取注册表，定位别名所在行
   ↓ 若不存在则提示中止
3. 从注册表中删除该行
   ↓
4. 从项目组project.md中删除对应子项目段落
   ↓
5. 输出摘要
   ↓
6. 提示用户可手动检查或清理磁盘文件
```

### 安全规则

| 规则 | 说明 |
|------|------|
| ❌ 绝对不执行删除命令 | 不执行 `rm`/`del` 等删除文件命令 |
| ✅ 仅修改.md文本 | 只移除注册表行和项目组project.md中的段落 |
| ✅ 实际文件不动 | 项目目录和project.md仍然保留 |

## 操作三：手动备份

### 触发词

```
@project-administrator 备份配置
@project-administrator 手动备份
```

### 执行流程

直接执行备份逻辑，不进行任何新增/删除操作。完成后报告备份路径。

---

## 备份逻辑详解

### 备份文件夹命名规则

1. 扫描 `{备份根目录}` 下所有子文件夹
2. 筛选格式为 `YYYYMMDD_NN` 的文件夹
3. 找到今天日期下的最大序号
4. 新序号 = 最大序号 + 1

### 备份目录结构

```
{备份根目录}/
{备份日期文件夹}/
├── .workbuddy/
│   ├── project-registry.md
│   └── skills/
│       └── [所有技能目录]
├── [其他配置文件目录]/
```

## 交付物与注意事项

### 成果说明

本技能的交付物是**配置变更 + 完整备份**：
- 注册表更新 / 项目组定义更新
- 完整配置备份到备份目录

### 适用场景

✅ **最佳适用**：
- 新项目启动时的注册
- 项目废弃时的注销（仅移除注册）
- 定期配置备份

❌ **不适用**：
- 只想查看配置（使用project-config-editor）
- 只想修改白名单（使用project-config-editor）

### 使用注意事项

| 注意事项 | 说明 |
|---------|------|
| ⚠️ 备份是强制前置 | 任何新增/删除操作前必须先备份，不允许跳过 |
| ⚠️ 别名不能重复 | 新增时如果别名已存在会报错，需更换别名 |
| ⚠️ 删除仅移除注册 | 不会删除任何磁盘文件，误删后可从备份恢复 |
| ⚠️ 备份完整性验证 | 建议备份后检查关键文件是否都存在 |
