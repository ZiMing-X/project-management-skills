# V9-Base 快速入门

> **适用版本**：V9-Base v2.9.0
> **前置要求**：WorkBuddy 已安装并可正常运行
> **最后更新**：2026-05-19

---

## 一、什么是 V9-Base

V9-Base 是 xmgl 项目管理治理框架的**最小可行集**，仅包含 5 个核心技能，覆盖项目最基本的生命周期骨架。

**与 V9-Standard (20技能) 和 V9-Full (27技能) 的差异**：V9-Base 只保留最核心的 5 个技能 —— 会话守护、项目追踪、任务门禁、收工协议、项目管理。适合轻量级使用。

---

| 分类 | 数量 | 包含技能 |
|:---|:--:|:---|
| 核心治理 | 5 | checkpoint-guard, project-manager, project-tracker, session-guard, shutdown-protocol |

## 三、5步快速上手

### 第1步：部署技能包

在 WorkBuddy 中输入：

```
请帮我把 V9-Base 源目录下的技能文件部署到 ~/.workbuddy/skills/
源路径：{你的V9-Base目录路径}
```

AI 会自动读取所有 5 个技能文件并安装到正确位置。

### 第2步：验证部署

重启 WorkBuddy，输入：

```
/skills
```

检查是否有 5 个技能显示在列表中。

### 第3步：会话守护 + 初始化项目

```
会话守护
```

激活 session-guard，然后初始化项目：

```
→ "初始化项目"
   AI：请输入项目别名
   你：myproj
   AI：请输入项目全称
   你：我的新项目
   AI：请输入项目根目录
   你：D:/projects/myproj
   AI：请输入白名单任务ID
   你：task-1
```

### 第4步：开始项目追踪

```
记录事件
项目：myproj
事件：完成 V9-Base 部署
标签：里程碑
```

### 第5步：收工（每次结束）

```
收工
```

激活 shutdown-protocol，将变更持久化到文件。

## 五、升级到 V9-Standard

### 从 V9-Base 升级

V9-Base (5技能) 升级到 V9-Standard (20技能) 增加的能力：
- 治理工具：automation-task-auditor, automation-verifier, problem-solution-standard, p10-archive-router, prompt-advisor, prompt-optimizer, skill-integration-merge
- 部署工具：install-skill-package, v9base-deploy, v9-release-checker, token-monitor
- 记忆会话：context-survival, session-manager, window-aware, workbuddy-debug-helper

## 六、每次会话的工作流

```
启动 WorkBuddy
    │
    ▼
会话守护（session-guard）  ← 检查状态
    │
    ▼
项目追踪（project-tracker） ← 记录进展
    │
    ▼
任务门禁（checkpoint-guard）← 复杂任务断点
    │
    ▼
收工（shutdown-protocol）   ← 持久化变更
```

## 七、常见问题

**Q：V9-Base 够用吗？**
A：V9-Base 覆盖最基本的项目治理需求。如果日常使用频繁，建议升级到 V9-Standard。

**Q：可以升级到 V9-Full 吗？**
A：可以。升级路径：V9-Base → V9-Standard → V9-Full。
