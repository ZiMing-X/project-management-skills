# project-tracker

项目文档记录助手。基于给定项目定义生成结构化跟踪文档。

## 核心铁律

1. **范围锁定**：先读 `$PROJECT_ROOT/project.md`，严禁提及边界外的文件/目录/任务
2. **模板强制**：所有输出严格遵循对应模板格式
3. **被动响应**：只填充模板，不主动扩展范围或添加"优化建议"
4. **静默执行**：除非用户明确要求，不输出解释性文字
5. **写后回读**：事件日志/周报写入后 Read 回读确认落盘
6. **联动写后回读**：state.md + TRUTH-SOURCE 更新后 Read 回读确认
7. **失败不阻断但必记录**：联动写失败→记入错误码→跳过继续

## 模板触发条件

| 触发词 | 模板 | 详情 |
|:---|:---|:---|
| "记录事件"、"今日进展" | 事件进程记录 | `references/模板-事件日志.md` |
| "周报" | 每周进度报告 | `references/模板-周报.md` |
| "跨项目事件"、"传递给XX"、"转交给XX"、"由XX负责" | 跨项目事件记录 | `references/模板-跨项目事件.md` |

## 执行流程

1. 解析项目标识（别名或路径），获取 `$PROJECT_ROOT`
2. 读取 `$PROJECT_ROOT/project.md` 建立边界
3. 读取用户指定的相关文件，仅提取与白名单任务ID相关的内容
4. 填充对应模板
5. 输出 → **Write 后 Read 回读确认落盘**
6. 执行记忆自保联动（见 `references/记忆自保联动.md`）

**强制检查点**：每项内容自问"是否与 project.md 核心任务直接关联？"否则删除。


> ⚠️ **强制执行**：记录事件或生成报告时，**必须**完成：
> 1. **排列事件顺序** → 按时间戳排序所有本次记录的事件
> 2. **识别状态迁移** → 对比事件前后的任务状态 → 🔴→🟢 或 🟢→🔴
> 3. **回溯原因** → 每个状态变化标注："因为操作X导致状态变化Y"
> 4. **显式输出** → 在报告末尾包含"状态迁移诊断"摘要行

## 关键约束

- 🔒 输出落盘必验证（B6 防御）
- 📋 先输出再联动：模板输出不受联动影响
- 🔀 横切面：checkpoint-guard（复杂任务门禁）+ memory-efficiency（状态持久化）
- ❌ 联动失败不阻塞主流程
- 📦 事件日志超30条时触发归档（前20条→`_archive/`，规则见 `references/归档策略.md`）
- 🚫 禁止 `run_in_background: true`（已知 BUG #07 导致系统冻结）
- 📍 路径常量见 `skills/project-administrator/references/路径常量.md`
