# automation-task-auditor — 自动化任务审查与优化

对 WorkBuddy 自动化任务进行分组识别、深度分析、文件提取归档和文档规范化。

## 适用场景

- 查看自动化任务分组 | 分析系列任务执行顺序/调度冲突/依赖
- 提取任务解析文档和关联脚本到指定目录 | 规范化命名、识别冗余

## 四阶段流程概览

| 阶段 | 操作 | 触发条件 | 产出 |
|:---|:-----|:--------|:-----|
| Phase 1 | 分组识别 — SQLite 查询 + 聚类分组 | 未指定系列 | 分组表 |
| Phase 2 | 深度分析 — 时间线 + 冲突检测 + 优化点 | 已指定系列 | 分析报告 |
| Phase 3 | 文件提取 — 复制解析文档 + 脚本 + 生成清单 | 需要提取 | 归档目录 |
| Phase 4 | 文档规范 — 重命名 + 更新清单 | Phase 3 完成后 | 规范化目录 |

> **详细 Python 代码 + 分组规则** → `references/phase详情.md`
> **数据源参考**（DB 结构/RRule/目录路径）→ `references/field-guide.md`

## 工作流程决策树

```
用户触发技能
  │
  ├─ 未指定系列 → Phase 1 分组识别 → 展示分组表 → 用户选择系列
  │
  ├─ 已指定系列 → Phase 2 深度分析
  │                     │
  │                     ├─ 仅分析 → 输出分析报告 → 结束
  │                     │
  │                     └─ 需要提取 → Phase 3 文件提取 → Phase 4 文档规范 → 结束
  │
  └─ 用户说"全部任务" → Phase 1 → 对每个系列执行 Phase 2
```

## 关键约束

1. **只复制不移动** — 所有文件操作为 copy，源文件不变
2. **数据库为准** — 任务定义以 SQLite 数据库为准
3. **嵌套问题** — `02_任务解析` 目录存在嵌套重复，始终取最干净版
4. **脚本验证** — 提取脚本前必须验证源路径存在
5. **用户确认** — Phase 3 执行前展示计划等待确认
6. **Phase 2 优化点检查**：时间冲突 | order 不一致 | 冗余任务 | 兜底定位 | 脚本缺失 | 工作目录漂移


> 执行本技能时，必须进行时序推理：
> 1. 解析 rrule 时，**按时间排序任务**，建立🟢（正常调度）和🔴（冲突/遗漏）标记
> 2. 对比各任务的 `created_at`/`updated_at`，判断是否有**长期未更新的僵尸任务**
> 3. 如发现同一时间段多个 ACTIVE 任务，输出**时间冲突🔴**，分析资源竞争
> 4. 分析结束后，输出**时间线状态摘要**：`🟢 正常 N个 | 🟡 需关注 M个 | 🔴 冲突 K个`
