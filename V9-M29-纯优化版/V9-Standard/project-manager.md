# project-manager — 通用项目管理

通过对话交互获取项目信息，创建标准化项目文档骨架（PROJECT.md / CHANGELOG.md / ops/）并跟踪全生命周期进度。

## 参数化设计

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 项目名称 | 用于文档标题 | 用户输入 |
| 项目代号 | 简称 | 用户输入 |
| 项目根目录 | 绝对路径 | 当前工作目录 |
| 核心需求 | R1~R4 可自定义 | 从对话提取 |
| 模块列表 | 模块ID、任务、状态 | 初始为空 |

## 工作流程

```
初始化 → 添加模块 → 更新状态 → 创建操作指令 → 报告
```

## 核心操作

| # | 操作 | 触发词 | 说明 |
|:--|:-----|:------|:-----|
| 1 | 初始化项目骨架 | "初始化项目" | 创建 README/PROJECT/CHANGELOG/docs/ops 目录结构 |
| 2 | 添加/更新模块 | "添加模块 M2: ..." | 更新 PROJECT.md 模块表格 |
| 3 | 更新模块状态 | "M1.2完成了" | 更新状态 + 自动推进"下一个行动" |
| 4 | 添加自定义日志 | "记录一下"/"记个里程碑" | CHANGELOG.md 顶部追加 |
| 5 | 生成操作指令 | "生成操作指令"/"创建 ops" | 生成 ops/op-XX-{名称}.md 模板 |
| 6 | 显示项目状态 | "项目进度"/"汇报进度" | 读取 PROJECT.md 输出摘要 |

> **详细步骤** → `references/操作详情.md`
> **文件模板**（README/PROJECT/CHANGELOG/docs/ops）→ `references/文件模板.md`

## 行为准则

- **日期替换** — `{{当前日期}}` 替换为 YYYY-MM-DD
- **文件保护** — 创建前检查是否存在，绝不覆盖
- **格式对齐** — PROJECT.md 表格保持 Markdown 完美对齐
- **日志连续** — CHANGELOG.md 顶部追加，时间倒序
- **及时确认** — 每次文件操作后输出简短确认


> 执行本技能时，必须进行时序推理：
> 1. 读取 PROJECT.md 时，**按"最后更新"列排序**，识别长期未更新的模块
> 2. 更新状态时，**对比前后状态时间**，判断是否有跳跃式进展
> 3. 推进"下一个行动"时，**按模块优先级 + 最后更新时间**排序，优先推进最久未动的模块
