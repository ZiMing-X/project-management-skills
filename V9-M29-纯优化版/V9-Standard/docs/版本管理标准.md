# 产品文档版本管理标准

> 本文档版本：v2.9.0 | 适用于 xmgl项目管理技能体系 V9+

**功能**：规定xmgl项目管理技能体系文档纳入版本体系的标准要求

---

## 1. 文档与体系版本对应规则

### 1.1 版本同步原则

| 规则 | 说明 |
|------|------|
| **主版本绑定** | 产品文档主版本号与 xmgl体系主版本号同步 |
| **次版本联动** | 体系次版本升级时，文档必须同步更新 |
| **修订版独立** | 文档的修订号可在不依赖体系版本的情况下独立递增 |
| **强制一致** | 所有文档的版本号必须与体系版本严格一致 |

### 1.2 版本号对应表

| xmgl体系版本 | 对应文档版本 | 快照包 |
|--------------|-------------|--------|
| v2.9.0 | v2.9.0 | V9-2026-05-17 |
| v10.0.0 | v10.0.0 | V10-2026-05-16 |
| v2.4.0 | v1.1.0 | V8-2026-05-12 |

### 1.3 文档版本号格式

```
文档版本格式：v主版本.次版本.修订号

示例：v2.9.0
- 主版本(2)：与xmgl体系主版本绑定，重大架构变更时递增
- 次版本(0)：与xmgl体系次版本绑定，功能新增时递增
- 修订号(0)：文档内容修订时独立递增
```

### 1.4 版本强制一致性要求

| 检测点 | 文件位置 | 必须一致 |
|--------|---------|:--------:|
| 体系版本 | TRUTH-SOURCE.md | ✅ |
| 技能全景图 | 00-技能全景图.md | ✅ |
| 根目录 | README.md | ✅ |
| 文档标准 | docs/版本管理标准.md | ✅ |
| 版本管理 | docs/版本管理维护指南.md | ✅ |

## 2. 文档变更触发条件

### 2.1 必须更新文档的情形

| 触发条件 | 影响的文档 |
|---------|-----------|
| 体系主版本升级 | 全部文档（重大变更） |
| 体系次版本升级 | 全部文档（功能变更） |
| 新增/删除技能 | 技能全景图、根目录README、STANDARD |
| 触发词变更 | 对应技能的SKILL.md |
| 工作流程变更 | 相关技能文档 |
| 新增插件 | 对应技能的SKILL.md |

### 2.2 建议更新文档的情形

| 触发条件 | 影响的文档 |
|---------|-----------|
| 优化操作步骤 | 相关技能文档 |
| 新增使用场景 | 相关技能文档 |
| 发现文档错误 | 相关文档 |

## 3. 文档维护流程

### 3.1 标准维护流程

```
体系版本变更
     │
     ▼
评估影响范围 → 确定需更新的文档
     │
     ▼
执行文档更新 → 按本文档标准执行
     │
     ▼
更新文档版本号
     │
     ▼
执行文档内检（读取验证无乱码）
     │
     ▼
提交并记录 CHANGELOG
```

### 3.2 文档更新检查清单

每次文档更新必须执行：

- [ ] 确认更新的文档范围
- [ ] 更新文档内的版本号（页眉和页脚）
- [ ] 更新 `最后更新` 日期
- [ ] **读取文档验证编码正确（无乱码）**
- [ ] 检查文档内链接是否有效
- [ ] 更新 `CHANGELOG.md` 中的对应条目
- [ ] 确认所有文档版本号一致
- [ ] 如需创建快照，执行快照归档

## 4. 文档质量标准

### 4.1 完整性标准

| 标准 | 要求 |
|------|------|
| 必需章节 | 每个技能文档必须包含：定位、核心能力、触发词 |
| 交叉引用 | 相关技能必须相互链接 |
| 示例 | 操作类文档必须包含可执行示例 |

### 4.2 准确性标准

| 标准 | 要求 |
|------|------|
| 命令验证 | 所有命令必须在实际环境验证通过 |
| 路径正确 | 所有路径必须使用正确格式 |
| 版本对应 | 文档内容必须与声明的版本匹配 |
| **编码正确** | 文件必须为UTF-8编码，读取无乱码 |

### 4.3 编码质量强制检查

| 检查项 | 方法 | 判定标准 |
|-------|------|---------|
| 读取测试 | Read工具读取文件 | 无乱码显示 |
| 内容完整性 | 对比快照 | 关键章节齐全 |
| 触发词有效性 | 实际触发测试 | 可正常激活技能 |

## 5. CHANGELOG 管理

### 5.1 CHANGELOG 更新时机

| 变更类型 | 是否需要记录 |
|---------|------------|
| 主版本升级 | ✅ 必须 |
| 次版本升级 | ✅ 必须 |

| 错别字修正 | ❌ 不需要 |
| 格式调整 | ❌ 不需要 |

## 7. 版本强制一致性机制

### 7.1 B6防御层（编码损坏防护）

|-------|------|
