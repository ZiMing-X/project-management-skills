## 一、历史版本管理

> **机制**：VERSIONS/ 目录按版本存放历史快照。

### 目录结构

```
V9-M29-纯优化版/
├── V9-Base/              ← 5核心技能（最小可行集）
├── V9-Standard/          ← 本文档所在（20技能日常推荐）
│   ├── 00-xmgl框架总览.md
│   ├── 00-技能全景图.md
│   ├── README.md
│   ├── STANDARD.md
│   ├── config.env.example
│   ├── *.md               ← 20个技能文件
│   ├── docs/
│   │   ├── 快速入门.md
│   │   ├── 版本管理标准.md
│   │   └── 版本管理维护指南.md
├── V9-Full/              ← 27技能全量版
├── release/               ← 发布包
└── README.md              ← V9根README
```

### 快照规则

| 规则 | 说明 |
|------|------|
| **触发时机** | Y（次版本）升级时**必须**创建快照；Z（修订版）升级时可选 |
| **命名格式** | `V9-YYYY-MM-DD/` |
| **内容** | 完整复制当次版本时的所有技能文件 |
| **只读** | 快照目录中的文件不得修改，仅供追溯查阅 |

## 二、V9-Standard 技能详解（20个）

### 项目治理层（5个）

#### 01 project-tracker（项目文档记录）

| 属性 | 说明 |
|------|------|
| **定位** | 项目事件日志、周报、待办清单 |
| **核心能力** | 事件进程记录、每周进度报告、项目级待办管理 |
| **安全边界** | 严格限定在 project.md 定义的项目范围内 |

**触发词**：`记录事件`、`今日进展`、`周报`、`汇报进度`

---

#### 02 project-manager（项目管理）

| 属性 | 说明 |
|------|------|
| **定位** | 项目文档体系创建与维护 |
| **核心能力** | 初始化项目、模块状态推进、进度更新 |

**触发词**：`初始化项目`、`添加模块`、`任务完成汇报`、`汇报进度`

---

#### 03 shutdown-protocol（收工协议）

| 属性 | 说明 |
|------|------|
| **定位** | 会话结束的7步持久化协议（记忆自保体系写入端） |
| **核心能力** | 扫描变更→更新 state.md → 更新 project.md → 更新 TRUTH-SOURCE → 写记忆 → 清 checkpoint |

**触发词**：`收工`、`shutdown`、`wrap up`、`结束会话`

---

#### 04 session-guard（会话守护）

| 属性 | 说明 |
|------|------|
| **定位** | 会话启动守护（记忆自保体系读取端） |
| **核心能力** | 读取 TRUTH-SOURCE 权威快照，SHA256 指纹校验，检测上下文压缩漂移并纠偏 |
| **并发安全** | SHA256 读前/写回双校验，字段级 .lock 并发控制 |

**触发词**：`会话守护`、`检查记忆`、`状态审计`、`session guard`

---

#### 05 checkpoint-guard（任务门禁）

| 属性 | 说明 |
|------|------|
| **定位** | 复杂任务（>5步）强制前置门禁 |
| **核心能力** | 创建 nav → 同步 checkpoint_ptr → 每步更新断点 → 24h 未更新告警 |
| **目录安全** | .dir_snapshot.txt 事前/事后 diff 校验 |

**触发词**：`任务门禁`、`checkpoint`、`断点续传`

---
