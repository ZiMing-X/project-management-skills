# checkpoint-guard — 强制门禁·断点保护

执行层强制门禁。复杂任务启动前必须先建 Checkpoint，不通过则阻止。核心原则：**先建 Checkpoint 再执行，不跳过不绕行**。

## 触发条件

| 条件 | 说明 |
|:---|:---|
| 预估步骤 > 5 步 | 工具调用密度过高风险 |
| 用户说"复杂任务""多步骤""分步执行" | 显式声明复杂度 |
| 涉及跨会话执行 | 断联风险 |

## 强制检查清单 + 拦截铁律

| # | 检查项 | 拦截铁律 |
|:--|:---|:---|
| 1 | 🟢 已创建导航视图文件 `TASK-{id}-nav.md` | 断点指针必须在导航视图 |
| 2 | 🟢 已创建当前步骤执行工单 | 当前+下一步工单同时加载，不加载全部 |
| 3 | 🟢 断点指针已同步到 TRUTH-SOURCE.md | 中间结果只存引用路径，不存全文 |
| 4 | 🟢 已确认每步输出物存放路径 | 完成一步后，上一步工单立即降级到 L3 |

**只有全部通过，才能进入执行。** 任一未通过 → 🔴 拦截。

## 导航视图模板

```markdown
# 导航视图 | 任务: {任务名} | 状态: 执行中

| Step | 状态 | 关键产出 | 断点指针 |
|:----:|:----:|:---------|:--------:|
| 1 | ⏳ | — | ← 起点 |
| 2 | ⏳ | — | |

**目标**: {一句话}
**全局约束**: {只读/安全限制}
**异常处理**: 任一步骤 > 3次重试 → 暂停并通知用户
```

## 导航视图约束使用指南

### 何时需要设置导航约束

| 场景 | 说明 | 示例 |
|:---|:---|:---|
| **多分支并行** | 同一任务拆分多个子任务，各自独立执行 | P-09 和 P-10 并行推进 |
| **跨会话断点续传** | 任务中断后恢复，需定位上次进度 | 上下文压缩后重新连接 |
| **依赖链任务** | 后续步骤依赖前序步骤产出 | 数据提取 → 分析 → 报告 |
| **共用资源冲突** | 多个步骤操作同一文件/目录 | SKILL.md 被多个修复步骤修改 |

### 约束声明格式

在导航视图模板中增加 `约束` 行：

```markdown
**约束**:
- 类型: {parallel | sequential | exclusive}
- 冲突资源: {文件路径或资源名}
- 优先级: {步骤序号} 最高
```

| 约束类型 | 含义 | 典型场景 |
|:---|:---|:---|
| `parallel` | 步骤可并行执行 | 独立子任务 |
| `sequential` | 必须严格按顺序执行 | 有产出依赖 |
| `exclusive` | 独占资源，其他步骤禁止触碰 | 文件编辑冲突 |

### 全局约束字段填写指导

导航视图模板中的 `**全局约束**` 字段用于声明整个任务的执行边界：

| 字段值 | 含义 | 适用场景 |
|:---|:---|:---|
| `只读操作` | 仅允许读取文件，禁止任何写入/编辑/删除 | 调研、代码审查、安全审计 |
| `只读 + 搜索` | 在上述基础加上网络搜索 | 技术调研、文档搜索 |
| `编辑操作需确认` | 每次编辑前需用户显式批准 | 高风险文件的修改 |
| `安全模式` | 禁止访问外部网络，禁止文件删除 | 涉密项目 |
| `无限制`（默认） | 无特殊限制 | 常规开发任务 |

**示例**：
```markdown
**全局约束**: 只读操作
**全局约束**: 只读 + 搜索
**全局约束**: 编辑操作需确认
```

> **注意**：全局约束是任务级声明，不同于 Step 级的 `约束`（parallel/sequential/exclusive）资源冲突声明。

### 冲突解决优先级

```
规则1: 低序号步骤 > 高序号步骤（步骤1优先于步骤2）
规则2: exclusive > sequential > parallel（独占优先于顺序优先于并行）
规则3: 同级冲突 → 暂停并通知用户裁决
规则4: 超时等待 30s 后自动降级为 sequential
```

## 🔒 并发安全增强（批次7 P-040，v3.1.0）

### 目录状态快照协议

**任何涉及以下操作前，必须执行目录快照：**
- 批量文件删除/移动（涉及 ≥3 个文件）
- 递归目录操作（Remove-Item -Recurse / rm -rf）
- 归档操作（移动文件到 archive/）
- 目录内容扫描（当扫描结果用于后续决策时）

```
Step 1: 锁定目录
  touch <target_dir>/.dir.lock

Step 2: 生成快照
  ls -laR <target_dir> > <target_dir>/.dir_snapshot.txt
  → 快照包含：文件名 / 大小 / 修改时间 / 权限

Step 3: 执行操作
  按照快照清单逐项操作
  每操作一项后对比快照确认

Step 4: 操作后验证
  diff <(ls -laR <target_dir>) <(cat <target_dir>/.dir_snapshot.txt)
  确认变更符合预期

Step 5: 释放锁
  rm <target_dir>/.dir.lock
  mv <target_dir>/.dir_snapshot.txt <target_dir>/.dir_snapshot.done.txt
```

### 危险操作 .lock 约定

| 操作类型 | 锁文件路径 | 持有方 | 有效期 |
|:---|:---|:---|:---|
| 递归删除 | `<dir>/.del.lock` | 执行 agent | 操作完成即释放 |
| 批量移动 | `<dir>/.move.lock` | 执行 agent | 操作完成即释放 |
| 归档操作 | `archive/.archive.lock` | 执行 agent | 归档完成即释放 |
| 全量扫描 | `<dir>/.scan.lock` | 扫描 agent | 扫描完成即释放 |

### 冲突检测规则

```
规则1：操作前先检查 .lock 文件是否存在
规则2：.lock 存在 → 等待 5s 后重试 (最多 3 次)
规则3：3 次仍失败 → 放弃操作，输出 🟡 CG-005 "目录被锁定"
规则4：.del.lock 优先级最高，其他操作不得在 .del.lock 期间执行
```

### 新增错误码

| 错误码 | 说明 | 处理 |
|:---|:---|:---|
| 🟡 CG-005 | 目录被锁定 | 等待或提示用户手动解锁 |
| 🟡 CG-006 | 快照校验失败 | 变更不符合预期，暂停操作保留现场 |

### 危险删除操作强制清单（已在个人文件安全规则中定义）

```
⬛ 删除前必须：
  1. ls -laR <target> → 列出所有受影响文件
  2. 显示完整清单给用户 → 需用户显式确认
  3. 确认后执行 → 逐项删除（非批量）
  4. 删除后验证 → diff 快照确认
⬛ 禁止：静默删除 / 跳过清单 / 用户未确认直接执行
```


> 执行门禁时，必须进行时序推理：
> 1. 读取 TRUTH-SOURCE.md 的 `checkpoint_ptr`，**比较时间戳**判断是否为当前会话遗留
> 2. 若指针时间超过 24h → 🟡 CG-003 "可能过时"，**回溯上次操作的断点位置**
> 3. 导航视图中已完成步骤，**对比完成时间**判断是否需重做
